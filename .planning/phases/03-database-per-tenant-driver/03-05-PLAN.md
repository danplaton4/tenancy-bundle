   - Compiler pass that makes `doctrine.dbal.tenant_connection`, `tenancy.database_switch_bootstrapper`, and `doctrine.orm.tenant_entity_manager` public for test inspection.
   ```php
   declare(strict_types=1);
   namespace Tenancy\Bundle\Tests\Integration\Support;

   use Symfony\Component\DependencyInjection\Compiler\CompilerPassInterface;
   use Symfony\Component\DependencyInjection\ContainerBuilder;

   final class MakeDatabaseServicesPublicPass implements CompilerPassInterface
   {
       public function process(ContainerBuilder $container): void
       {
           $ids = [
               'doctrine.dbal.tenant_connection',
               'doctrine.orm.tenant_entity_manager',
               'doctrine.orm.landlord_entity_manager',
               'tenancy.database_switch_bootstrapper',
               'tenancy.context',
               'tenancy.bootstrapper_chain',
           ];
           foreach ($ids as $id) {
               if ($container->hasDefinition($id)) {
                   $container->getDefinition($id)->setPublic(true);
               } elseif ($container->hasAlias($id)) {
                   $container->getAlias($id)->setPublic(true);
               }
           }
       }
   }
   ```

1. Create `tests/Integration/DatabaseSwitchIntegrationTest.php`:
   - Uses `DoctrineTestKernel` with `setUpBeforeClass`/`tearDownAfterClass` for kernel lifecycle
   - In `setUpBeforeClass()`: boot kernel, create landlord DB schema (tenancy_tenants table), create two file-based SQLite DBs for tenants:
     - `$pathA = sys_get_temp_dir() . '/tenancy_test_tenant_a.db'`
     - `$pathB = sys_get_temp_dir() . '/tenancy_test_tenant_b.db'`
   - Use `Doctrine\ORM\Tools\SchemaTool` to create schema for TestProduct in both tenant DBs
   - In `tearDownAfterClass()`: delete all .db files, shutdown kernel

   Test methods:
   - `testSwitchToTenantAQueriesHitTenantADatabase()`:
     - Get TenantConnection from container: `$conn = self::$container->get('doctrine.dbal.tenant_connection')`
     - Call `$conn->switchTenant(['driver' => 'pdo_sqlite', 'path' => $pathA])`
     - Get tenant EM, persist a TestProduct('Product A'), flush
     - Assert: `SELECT COUNT(*) FROM test_products` on tenant connection returns 1
     - Switch to tenant B: `$conn->switchTenant(['driver' => 'pdo_sqlite', 'path' => $pathB])`
     - Get ManagerRegistry, call `resetManager('tenant')` to get fresh EM
     - Assert: `SELECT COUNT(*) FROM test_products` on tenant connection returns 0 (tenant B is empty)

   - `testSwitchToTenantBDoesNotSeeTenantAData()`:
     - Insert Product A in tenant A's DB, switch to tenant B
     - Reset EM, query TestProduct repository via tenant EM
     - Assert: findAll() returns empty array

   - `testLandlordEmIsUnaffectedByTenantSwitch()`:
     - Get landlord EM from container
     - Create and persist a Tenant entity in landlord DB
     - Switch to tenant A, switch to tenant B
     - Get landlord EM again, query for the Tenant entity
     - Assert: Tenant entity found (landlord EM unaffected by tenant switches)

   - `testSameTenantSwitchAfterResetReconnects()`:
     - Switch to tenant A, insert Product, flush
     - Reset connection, switch to tenant A again
     - Reset EM, query — product should still be there (same DB file)
