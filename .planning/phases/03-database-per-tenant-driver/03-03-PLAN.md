   - Add imports at top:

   - Change DoctrineTenantProvider EM argument from `service('doctrine.orm.default_entity_manager')` to use a conditional: when `tenancy.database.enabled` param exists, use `service('doctrine.orm.landlord_entity_manager')`; otherwise keep `service('doctrine.orm.default_entity_manager')`.
   - SIMPLEST APPROACH: Since `loadExtension` runs before `services.php` is imported, we cannot easily use parameter conditionals in a static services.php. Instead, move the conditional service registration INTO `loadExtension()`:

   Revised approach for `loadExtension()`:
   ```php
   if ($config['database']['enabled'] ?? false) {
       $services = $container->services();

       $services->set('tenancy.database_switch_bootstrapper', DatabaseSwitchBootstrapper::class)
           ->args([service('doctrine.dbal.tenant_connection')])
           ->tag('tenancy.bootstrapper');

       $services->set(EntityManagerResetListener::class)
           ->autoconfigure(true)
           ->args([service('doctrine')]);

       // Rewire DoctrineTenantProvider to landlord EM
       $builder->getDefinition('tenancy.provider')
           ->setArgument(0, new Reference('doctrine.orm.landlord_entity_manager'));
   }
   ```
   Note: The `$builder->getDefinition('tenancy.provider')` call works because `$container->import('../config/services.php')` runs first (at the top of `loadExtension`), so 'tenancy.provider' is already defined. Import `use Symfony\Component\DependencyInjection\Reference;` at the top of TenancyBundle.php.

   Add the required `use` statements to TenancyBundle.php:
   ```php
   use Symfony\Component\DependencyInjection\Reference;
   use Tenancy\Bundle\Bootstrapper\DatabaseSwitchBootstrapper;
   use Tenancy\Bundle\EventListener\EntityManagerResetListener;
   ```
   And use the function import:

4. Update `prependExtension()` to target landlord EM specifically instead of top-level `orm.mappings`:
   ```php
   public function prependExtension(ContainerConfigurator $container, ContainerBuilder $builder): void
   {
       $builder->prependExtensionConfig('doctrine', [
           'orm' => [
               'entity_managers' => [
                   'landlord' => [
                       'mappings' => [
                           'TenancyBundle' => [
                               'is_bundle' => false,
                               'type' => 'attribute',
                               'dir' => __DIR__.'/Entity',
                               'prefix' => 'Tenancy\\Bundle\\Entity',
                               'alias' => 'TenancyBundle',
                           ],
                       ],
                   ],
               ],
           ],
       ]);
   }
   ```
   IMPORTANT: This changes the prependExtension from targeting `orm.mappings` (top-level, applies to default EM) to targeting `orm.entity_managers.landlord.mappings` (explicit landlord EM). This ensures the Tenant entity is always mapped to the landlord EM regardless of which EM is set as default. Per Pitfall 4 in RESEARCH.md.

   CAVEAT: When `tenancy.database.enabled` is false (no dual-EM setup), the user has a single default EM, not a `landlord` named EM. The prependExtension should be conditional:
   - If `database.enabled` is true: prepend to `entity_managers.landlord.mappings`
   - If `database.enabled` is false: prepend to `orm.mappings` (current behavior, maps to default EM)

   To handle this, check the bundle config in prependExtension. BUT: `prependExtension` runs before `loadExtension`, so we don't have resolved config. Instead, read the raw config from the container:
   ```php
   public function prependExtension(ContainerConfigurator $container, ContainerBuilder $builder): void
   {
       $configs = $builder->getExtensionConfig('tenancy');
       $databaseEnabled = false;
       foreach ($configs as $config) {
           if (isset($config['database']['enabled'])) {
               $databaseEnabled = $config['database']['enabled'];
           }
       }

       if ($databaseEnabled) {
           $builder->prependExtensionConfig('doctrine', [
               'orm' => [
                   'entity_managers' => [
                       'landlord' => [
                           'mappings' => [
                               'TenancyBundle' => [
                                   'is_bundle' => false,
                                   'type' => 'attribute',
                                   'dir' => __DIR__.'/Entity',
                                   'prefix' => 'Tenancy\\Bundle\\Entity',
                                   'alias' => 'TenancyBundle',
                               ],
                           ],
                       ],
                   ],
               ],
           ]);
       } else {
           $builder->prependExtensionConfig('doctrine', [
               'orm' => [
                   'mappings' => [
                       'TenancyBundle' => [
                           'is_bundle' => false,
                           'type' => 'attribute',
                           'dir' => __DIR__.'/Entity',
                           'prefix' => 'Tenancy\\Bundle\\Entity',
                           'alias' => 'TenancyBundle',
                       ],
                   ],
               ],
           ]);
       }
   }
   ```
