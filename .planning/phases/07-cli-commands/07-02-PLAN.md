2. **TenantRunCommand.php** -- Create `src/Command/TenantRunCommand.php`:

   ```php
   <?php
   declare(strict_types=1);
   namespace Tenancy\Bundle\Command;

   use Symfony\Component\Console\Attribute\AsCommand;
   use Symfony\Component\Console\Command\Command;
   use Symfony\Component\Console\Input\InputArgument;
   use Symfony\Component\Console\Input\InputInterface;
   use Symfony\Component\Console\Output\OutputInterface;
   use Symfony\Component\Process\Process;
   use Tenancy\Bundle\Provider\TenantProviderInterface;
   ```

   Class structure:
   - `#[AsCommand(name: 'tenancy:run', description: 'Run a Symfony console command scoped to a specific tenant')]`
   - `final class TenantRunCommand extends Command`
   - Constructor params: `private readonly TenantProviderInterface $tenantProvider`, `private readonly string $projectDir`
   - `configure()`:
     - `addArgument('tenant', InputArgument::REQUIRED, 'Tenant slug')`
     - `addArgument('command_string', InputArgument::REQUIRED, 'The console command to run (e.g. "app:some-command arg1 arg2")')`
   - `execute(InputInterface $input, OutputInterface $output)`:
     a. `$tenantSlug = $input->getArgument('tenant')`
     b. `$commandString = $input->getArgument('command_string')`
     c. Validate tenant exists: `$this->tenantProvider->findBySlug($tenantSlug)` -- let TenantNotFoundException or TenantInactiveException bubble as non-zero exit
     d. Build console path: `$consolePath = $this->projectDir . '/bin/console'`
     e. Build process using `Process::fromShellCommandline()` for proper shell quoting:
        ```php
        $process = Process::fromShellCommandline(
            sprintf('%s %s %s --tenant=%s',
                escapeshellarg(\PHP_BINARY),
                escapeshellarg($consolePath),
                $commandString,
                escapeshellarg($tenantSlug)
            )
        );
        ```
     f. Set timeout to null (no timeout -- inner commands may be long-running)
     g. Stream output: `$process->run(function (string $type, string $buffer) use ($output): void { $output->write($buffer); })`
     h. Return `$process->getExitCode() ?? 0`

4. **TenantRunCommandTest.php** -- Create unit tests. Since `Process` is a concrete class and subprocess spawning is hard to unit-test, use a **process factory** injection pattern or test at the integration boundary:

   **Practical approach**: Make the Process creation injectable. Add an optional `?\Closure $processFactory = null` constructor parameter. In production, it defaults to null and the command creates the Process internally. In tests, inject a closure that returns a mock/stub Process.

   Actually, simpler: use CommandTester and mock TenantProviderInterface. The unit tests verify:
   - Argument parsing (tenant slug, command string)
   - Tenant validation via findBySlug (mock throws TenantNotFoundException)
   - For the actual Process execution: inject a `$processFactory` callable as the third constructor param (after projectDir). Signature: `callable(array $command): Process`. Default in DI: `null`. When null, command creates Process internally. When provided, calls factory.

   **Constructor**: `TenantProviderInterface $tenantProvider, string $projectDir, ?\Closure $processFactory = null`

   In services.php, do NOT pass processFactory (it defaults to null, using real Process).

   Test cases:
   - `testValidTenantSpawnsProcess`: inject processFactory that captures args and returns a Process mock with exit code 0. Assert args contain PHP_BINARY, consolePath, command parts, and `--tenant=acme`.
   - `testChildExitCodePropagated`: processFactory returns mock with exit code 42. Assert command returns 42.
   - `testOutputForwarded`: processFactory returns mock that calls the output callback with "hello world". Assert output contains "hello world".
   - `testNonexistentTenantThrows`: mock TenantProviderInterface::findBySlug to throw TenantNotFoundException. Assert the exception propagates (command does not catch it).

   For the Process mock: create a simple stub. Process is not final, so PHPUnit can mock it. Configure `run()` to accept a callback, `getExitCode()` to return the configured value.
