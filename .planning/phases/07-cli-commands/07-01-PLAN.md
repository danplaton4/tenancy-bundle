   ```php
   /** @return TenantInterface[] */
   public function findAll(): array
   {
       /** @var TenantInterface[] $tenants */
       $tenants = $this->entityManager
           ->getRepository($this->tenantEntityClass)
           ->findAll();

       return $tenants;
   }
   ```
   This bypasses the cache intentionally -- `findAll()` is an operator tool (migration commands), not a hot path. Returns ALL tenants including inactive ones so operators have full visibility and can migrate inactive tenant DBs for reactivation.

1. **TenantMigrateCommand.php** -- Create `src/Command/TenantMigrateCommand.php`:

   ```php
   <?php
   declare(strict_types=1);
   namespace Tenancy\Bundle\Command;

   use Doctrine\DBAL\Connection;
   use Doctrine\Migrations\Configuration\Configuration;
   use Doctrine\Migrations\Configuration\Connection\ExistingConnection;
   use Doctrine\Migrations\Configuration\Migration\ExistingConfiguration;
   use Doctrine\Migrations\DependencyFactory;
   use Doctrine\Migrations\MigratorConfiguration;
   use Symfony\Component\Console\Attribute\AsCommand;
   use Symfony\Component\Console\Command\Command;
   use Symfony\Component\Console\Input\InputInterface;
   use Symfony\Component\Console\Input\InputOption;
   use Symfony\Component\Console\Output\OutputInterface;
   use Symfony\Component\Console\Style\SymfonyStyle;
   use Tenancy\Bundle\Bootstrapper\BootstrapperChain;
   use Tenancy\Bundle\Context\TenantContext;
   use Tenancy\Bundle\Provider\TenantProviderInterface;
   use Tenancy\Bundle\TenantInterface;
   ```

   Class structure:
   - `#[AsCommand(name: 'tenancy:migrate', description: 'Run Doctrine migrations for all tenants')]`
   - `final class TenantMigrateCommand extends Command`
   - Constructor params: `TenantProviderInterface $tenantProvider`, `BootstrapperChain $bootstrapperChain`, `TenantContext $tenantContext`, `string $driver`, `Connection $tenantConnection`, `Configuration $migrationsConfig`
   - `configure()`: Add `--tenant` option (InputOption::VALUE_OPTIONAL, 'Run migrations for a single tenant only')
   - `execute()`:
     a. If `$this->driver === 'shared_db'`: write to `$output->getErrorOutput()` the message `tenancy:migrate is only available with the database_per_tenant driver.` and return `Command::FAILURE`
     b. If `--tenant` option is provided: call `$this->tenantProvider->findBySlug($slug)` to get single tenant, set `$tenants = [$tenant]`
     c. Otherwise: `$tenants = $this->tenantProvider->findAll()`
     d. If `$tenants` is empty: `$io->writeln('No tenants found.')` and return `Command::SUCCESS`
     e. Iterate tenants, for each:
        - try: `$this->runMigrationsForTenant($tenant, $io)`, then `$io->writeln(sprintf(' <info>✓</info> %s', $tenant->getSlug()))`
        - catch `\Throwable $e`: append to `$failures[]`, `$io->writeln(sprintf(' <error>✗</error> %s (%s)', $tenant->getSlug(), $e->getMessage()))`
        - finally: `$this->tenantContext->clear()`, `$this->bootstrapperChain->clear()`
     f. Print summary: `Completed: {N} succeeded, {M} failed`
     g. If failures not empty, print failure list
     h. Return `Command::FAILURE` if `$failures` is not empty, else `Command::SUCCESS`
   - `runMigrationsForTenant(TenantInterface $tenant, SymfonyStyle $io)`:
     a. `$this->tenantContext->setTenant($tenant)`
     b. `$this->bootstrapperChain->boot($tenant)`
     c. Build `DependencyFactory::fromConnection(new ExistingConfiguration($this->migrationsConfig), new ExistingConnection($this->tenantConnection))`
     d. `$dependencyFactory->getMetadataStorage()->ensureInitialized()`
     e. `$planCalculator = $dependencyFactory->getMigrationPlanCalculator()`
     f. `$plan = $planCalculator->getPlanUntilVersion($dependencyFactory->getVersionAliasResolver()->resolveVersionAlias('latest'))`
     g. If `$plan->isEmpty()`: return early (already up to date)
     h. `$dependencyFactory->getMigrator()->migrate($plan, new MigratorConfiguration())`

For the migrations part, the unit test creates a `TestTenantMigrateCommand` that extends TenantMigrateCommand and overrides `runMigrationsForTenant` to track calls and optionally throw. Wait -- the class is `final`. So instead: inject a callable `$migrationRunner` as a constructor parameter (callable receives TenantInterface, throws on failure). In production, this callable wraps the DependencyFactory logic. In tests, it's a simple closure.

In `execute()`, call `($this->migrationRunner)($tenant)` instead of `$this->runMigrationsForTenant()`. The DI wiring in services.php passes an inline service or factory that builds the DependencyFactory closure. Actually -- this adds complexity to DI. Better approach: keep `runMigrationsForTenant` as a private method, and in unit tests, mock the `Connection` and `Configuration` to be no-ops.
