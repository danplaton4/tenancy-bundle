    **setUpBeforeClass():** Boot `BootstrapperTestKernel` (if not already shared with DoctrineBootstrapperIntegrationTest — use a separate static instance or cache_dir to avoid conflicts). Create schema for Tenant entity.

    **setUp():** Get `cache.app` from container, get `TenantContext` from container. Clear any leftover cache state.

    **tearDownAfterClass():** Shutdown kernel.

    Test methods:

    **testCacheAppIsDecoratedByTenantAwareAdapter():**
    - Get `cache.app` from container
    - Assert it is an instance of `TenantAwareCacheAdapter`

    **testCacheIsolationBetweenTenants():**
    - Create two Tenant entities (or stubs): tenantA (slug: 'alpha'), tenantB (slug: 'beta')
    - Set tenantA on TenantContext
    - Write a cache item: `$pool->getItem('greeting')` -> set('hello-alpha') -> `$pool->save($item)`
    - Verify: `$pool->getItem('greeting')->isHit()` is true, value is 'hello-alpha'
    - Switch to tenantB: `$tenantContext->clear(); $tenantContext->setTenant($tenantB)`
    - Verify: `$pool->getItem('greeting')->isHit()` is **false** (different namespace)
    - Write under tenantB: set('hello-beta'), save
    - Switch back to tenantA: clear, setTenant(tenantA)
    - Verify: `$pool->getItem('greeting')->get()` is still 'hello-alpha' (not overwritten by beta)

    **testClearTenantACacheDoesNotAffectTenantB():**
    - Set tenantA, write cache item 'key-a' = 'value-a', save
    - Set tenantB, write cache item 'key-b' = 'value-b', save
    - Set tenantA, call `$pool->clear()` — clears only tenantA's namespace
    - Set tenantA, verify `$pool->getItem('key-a')->isHit()` is false (cleared)
    - Set tenantB, verify `$pool->getItem('key-b')->isHit()` is **true** (unaffected)

    **testNoTenantDelegatesToGlobalPool():**
    - Do NOT set any tenant on TenantContext
    - Write a cache item 'global-key' = 'global-value', save
    - Verify: `$pool->getItem('global-key')->isHit()` is true, value is 'global-value'
    - Now set tenantA
    - Verify: `$pool->getItem('global-key')->isHit()` is **false** (different namespace — tenant namespace vs global)

    For tenant stubs in these tests, create a simple anonymous class implementing TenantInterface with the required methods, or persist real Tenant entities and load them. The anonymous class approach is simpler since we only need getSlug() to return the slug string.
