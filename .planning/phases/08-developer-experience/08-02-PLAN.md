Create `tests/Integration/Testing/InteractsWithTenancyTest.php`:

Namespace: `Tenancy\Bundle\Tests\Integration\Testing`

The test class extends `PHPUnit\Framework\TestCase` (not KernelTestCase) and manages its own kernel lifecycle via `setUpBeforeClass`/`tearDownAfterClass`, following the established integration test pattern (see `DoctrineBootstrapperIntegrationTest`). The trait requires `static::getContainer()` — since the class does NOT extend `KernelTestCase`, define a `static function getContainer()` helper method that returns `static::$kernel->getContainer()`.

IMPORTANT DESIGN: The trait's `tearDown()` calls `parent::tearDown()`. Since we extend `TestCase` directly (not KernelTestCase), `parent::tearDown()` is `TestCase::tearDown()` which is a no-op. This is fine — the trait works correctly.

However, the trait calls `static::getContainer()` which is a `KernelTestCase` static method. Since we extend `TestCase`, NOT `KernelTestCase`, we need to provide a compatible `getContainer()`. Define a `protected static function getContainer()` method that returns `static::$kernel->getContainer()`. This matches what KernelTestCase does internally.

**Class structure:**
```
use Tenancy\Bundle\Testing\InteractsWithTenancy;
use Tenancy\Bundle\Tests\Integration\Testing\Support\TenancyTestKernel;
use Tenancy\Bundle\Tests\Integration\Support\Entity\TestProduct;
use Tenancy\Bundle\Context\TenantContext;
use Doctrine\ORM\EntityManagerInterface;

final class InteractsWithTenancyTest extends TestCase
{
    use InteractsWithTenancy;

    private static TenancyTestKernel $kernel;

    public static function setUpBeforeClass(): void
    {
        // Clean up any leftover DB files from prior runs
        foreach ([
            sys_get_temp_dir() . '/tenancy_testing_trait_landlord.db',
            sys_get_temp_dir() . '/tenancy_testing_trait_placeholder.db',
        ] as $path) {
            if (file_exists($path)) {
                unlink($path);
            }
        }

        static::$kernel = new TenancyTestKernel();
        static::$kernel->boot();
    }

    public static function tearDownAfterClass(): void
    {
        static::$kernel->shutdown();

        foreach ([
            sys_get_temp_dir() . '/tenancy_testing_trait_landlord.db',
            sys_get_temp_dir() . '/tenancy_testing_trait_placeholder.db',
        ] as $path) {
            if (file_exists($path)) {
                unlink($path);
            }
        }
    }

    protected static function getContainer(): \Symfony\Component\DependencyInjection\ContainerInterface
    {
        return static::$kernel->getContainer();
    }
}
```

**Test methods (6 total, covering DX-01a through DX-01f):**

1. **`testInitializeTenantBootsContextAndSchema()`** (DX-01a):
   - Call `$this->initializeTenant('acme')`
   - Assert `TenantContext::hasTenant()` is true
   - Assert `TenantContext::getTenant()->getSlug()` is `'acme'`
   - Get the tenant EM via `static::getContainer()->get('doctrine')->getManager('tenant')`
   - Persist a `new TestProduct('Widget')` via the tenant EM, flush
   - Assert the product was inserted by fetching via `$em->getRepository(TestProduct::class)->findAll()` and checking count is 1
   - This proves both context activation AND schema creation on :memory: SQLite

2. **`testTearDownClearsContextAfterTest()`** (DX-01b):
   - Call `$this->initializeTenant('beta')`
   - Assert context has tenant 'beta'
   - Manually call `$this->tearDown()` (simulates PHPUnit calling it)
   - After tearDown, get a fresh TenantContext reference: `static::getContainer()->get('tenancy.context')`
   - Assert `hasTenant()` is false
   - NOTE: We cannot easily test "tearDown runs on exception" in an automated test because PHPUnit catches the exception and calls tearDown automatically. The manual verification in VALIDATION.md covers this edge case. This test proves the tearDown->clearTenant pathway works.

3. **`testTwoMethodsGetIsolatedDatabases()`** (DX-01c):
   - Call `$this->initializeTenant('tenant_x')`
   - Get tenant EM, persist `new TestProduct('X Product')`, flush
   - Call `$this->clearTenant()` (simulate between-method cleanup)
   - Call `$this->initializeTenant('tenant_y')`
   - Get tenant EM (fresh after resetManager), query `TestProduct::class` findAll
   - Assert count is 0 — tenant_y's :memory: SQLite has no data from tenant_x
   - Persist `new TestProduct('Y Product')` via tenant_y EM, flush
   - Assert count is 1 — only tenant_y's data

4. **`testAssertTenantActivePassesWithCorrectSlug()`** (DX-01d):
   - Call `$this->initializeTenant('acme')`
   - Call `$this->assertTenantActive('acme')` — should NOT throw
   - This test passes if no assertion failure is raised

5. **`testAssertNoTenantPassesWhenContextIsEmpty()`** (DX-01e):
   - Ensure no tenant is active (call `$this->clearTenant()` just in case)
   - Call `$this->assertNoTenant()` — should NOT throw
   - This test passes if no assertion failure is raised

6. **`testGetTenantServiceReturnsServiceFromContainer()`** (DX-01f):
   - Call `$this->initializeTenant('acme')`
   - Call `$service = $this->getTenantService(TenantContext::class)`
   - Assert `$service instanceof TenantContext`
   - Assert `$service->hasTenant()` is true (context was already initialized)

Each test method should call `$this->clearTenant()` in its body when it needs to reset state mid-test (tests 2 and 3). The trait's `tearDown()` handles cleanup between test methods automatically.

IMPORTANT: For test 2, after calling `$this->tearDown()`, the kernel container may have been nullified by parent::tearDown() if extending KernelTestCase. Since we extend TestCase, this is NOT an issue — `static::$kernel` remains available.
