Create `src/Testing/InteractsWithTenancy.php`:

Namespace: `Tenancy\Bundle\Testing`

Class-level docblock: Document that the trait is intended for `KernelTestCase` subclasses. Note that the test kernel must have `database.enabled: true` with a `TenantConnection` wrapperClass on the tenant connection (database-per-tenant mode). Reference `TenancyTestKernel` as the recommended kernel for test classes.

**Methods (all `protected` visibility except tearDown which is also protected):**

1. **`initializeTenant(string $slug): void`** — The core DX method. Sequence per locked decision:
   ```
   $container = static::getContainer();
   
   // Step 1: Clear any prior tenant context
   $tenantContext = $container->get('tenancy.context');
   $tenantContext->clear();
   
   // Also clear bootstrapper chain from prior tenant
   $chain = $container->get('tenancy.bootstrapper_chain');
   $chain->clear();
   
   // Step 2: Swap tenant connection to fresh :memory: SQLite
   $conn = $container->get('doctrine.dbal.tenant_connection');
   $conn->switchTenant(['driver' => 'pdo_sqlite', 'memory' => true]);
   
   // Step 3: Reset tenant EM + create schema
   $registry = $container->get('doctrine');
   $em = $registry->resetManager('tenant');
   $schemaTool = new \Doctrine\ORM\Tools\SchemaTool($em);
   $schemaTool->createSchema($em->getMetadataFactory()->getAllMetadata());
   
   // Step 4: Activate synthetic tenant in context
   $tenant = new \Tenancy\Bundle\Entity\Tenant($slug, $slug);
   $tenantContext->setTenant($tenant);
   
   // Step 5: Run all registered bootstrappers
   $chain->boot($tenant);
   ```

   IMPORTANT: Use `'memory' => true` (DBAL 4 syntax for in-memory SQLite) not `'path' => ':memory:'`. Both work but `memory => true` is the canonical DBAL 4 way. If this causes issues during testing, fall back to `'path' => ':memory:'`.

2. **`clearTenant(): void`** — Clears context and bootstrapper chain:
   ```
   $container = static::getContainer();
   $tenantContext = $container->get('tenancy.context');
   if ($tenantContext->hasTenant()) {
       $chain = $container->get('tenancy.bootstrapper_chain');
       $chain->clear();
   }
   $tenantContext->clear();
   ```
   Guard on `hasTenant()` to avoid running bootstrapper clear when no tenant was initialized (e.g., test that never called `initializeTenant`).

3. **`tearDown(): void`** — Override. Must call `clearTenant()` before `parent::tearDown()`:
   ```
   protected function tearDown(): void
   {
       $this->clearTenant();
       parent::tearDown();
   }
   ```
   PHPUnit always runs tearDown even when setUp/test throws. Calling `clearTenant()` before `parent::tearDown()` ensures the container is still available.

4. **`assertTenantActive(string $expectedSlug): void`** — Assertion helper:
   ```
   $tenantContext = static::getContainer()->get('tenancy.context');
   $this->assertTrue($tenantContext->hasTenant(), 'Expected an active tenant but TenantContext is empty');
   $this->assertSame($expectedSlug, $tenantContext->getTenant()->getSlug(), sprintf('Expected tenant "%s" but got "%s"', $expectedSlug, $tenantContext->getTenant()->getSlug()));
   ```
   Uses `$this->assertTrue()` and `$this->assertSame()` — these come from PHPUnit `TestCase` which `KernelTestCase` extends.

5. **`assertNoTenant(): void`** — Assertion helper:
   ```
   $tenantContext = static::getContainer()->get('tenancy.context');
   $this->assertFalse($tenantContext->hasTenant(), 'Expected no active tenant but TenantContext has one');
   ```

6. **`getTenantService(string $class): object`** — Service accessor:
   ```
   return static::getContainer()->get($class);
   ```
   Simple convenience wrapper. Return type `object`. Add `@template T of object` / `@param class-string<T> $class` / `@return T` PHPDoc for IDE autocompletion.

No DI registrations needed — the trait is a PHP trait, not a Symfony service.
