<task type="auto">
  <name>Task 1: Create Dockerfile and docker-compose.yml</name>
  <files>Dockerfile, docker-compose.yml</files>
  <action>
Create a Dockerfile with:
- ARG PHP_VERSION=8.2
- FROM php:${PHP_VERSION}-cli-alpine
- Install system deps needed by Composer and extensions: git, unzip, zip (via apk)
- COPY --from=composer:2 /usr/bin/composer /usr/bin/composer
- WORKDIR /app
- No COPY of source code (mounted via volume)
- No ENTRYPOINT override — default to sh so commands work naturally

Create docker-compose.yml with:
- version not needed (Compose V2 default)
- Service named "php"
- build context "." with args: PHP_VERSION=${PHP_VERSION:-8.2}
- Volume mount: .:/app (bind mount project root)
- Volume mount: composer-cache:/root/.composer (named volume for Composer cache persistence)
- working_dir: /app
- No ports, no depends_on, no other services
- Top-level volumes section declaring composer-cache

The PHP_VERSION variable should be read from the shell environment, defaulting to 8.2 if unset. Usage pattern: PHP_VERSION=8.3 docker compose build --no-cache
  </action>
  <verify>
    <automated>docker compose config 2>&1 | head -30</automated>
  </verify>
  <done>docker compose config validates without errors; service "php" defined with build args and volume mounts</done>
</task>

<task type="auto">
  <name>Task 2: Create Makefile and update .gitignore</name>
  <files>Makefile, .gitignore</files>
  <action>
Create a Makefile with these targets (use TAB indentation, not spaces):

- test: docker compose run --rm php vendor/bin/phpunit
  (default target)
- install: docker compose run --rm php composer install
- update: docker compose run --rm php composer update
- shell: docker compose run --rm php sh
- build: docker compose build --no-cache
  (for rebuilding after PHP_VERSION change)
- clean: docker compose down -v --rmi local

Add a .PHONY line for all targets.

Add a comment header explaining usage:
  # Usage:
  #   make test                      — run tests (PHP 8.2 default)
  #   PHP_VERSION=8.3 make build test — rebuild for PHP 8.3 and test
  #   make shell                     — interactive shell in container
  #   make install                   — install Composer dependencies

Update .gitignore: no changes needed — Dockerfile, docker-compose.yml, and Makefile should be tracked. Verify .gitignore does not already exclude them.
  </action>
  <verify>
    <automated>make -n test 2>&1 && make -n install 2>&1 && make -n shell 2>&1</automated>
  </verify>
  <done>make -n (dry run) for test, install, shell all print the expected docker compose commands</done>
</task>

<task type="auto">

  <files></files>
  <action>
Run the following sequence to verify the full setup works:
1. docker compose build (builds PHP 8.2 image)
2. docker compose run --rm php composer install (installs dependencies)
3. docker compose run --rm php vendor/bin/phpunit (runs the test suite)
4. Verify all tests pass

If any step fails, diagnose and fix the Dockerfile or docker-compose.yml. Common issues:

- File permissions on vendor/ (Alpine runs as root by default, should be fine)
